eMMC Management

Rockpi eMMC 백업 및 복구 가이드

본 가이드는 Rockpi 장비의 eMMC 데이터를 완전히 백업하고, 개발용 데스크톱(Linux 환경)에서 rkdeveloptool을 활용하여 원복하는 과정을 단계별로 다룹니다.

작업 개요

Rockpi에 구성된 운영체제 및 데이터를 이미지(.img) 파일로 안전하게 보관한 후, 필요시 보드의 하드웨어 인터페이스(Maskrom Mode)를 조작하여 PC에서 강제 플래싱하는 마스터 이미지 배포 및 복구 프로세스입니다.

1

1단계: Rockpi 본체에서 eMMC 백업 진행

현재 안정적으로 작동 중인 Rockpi 본체에 직접 SSH 또는 직렬 터미널로 접속하여 eMMC 전체를 백업합니다.

준비 사항

  • Rockpi 본체에 마운트할 64GB 이상의 여유 공간이 있는 USB 메모리 (또는 외장 드라이브)
BASH - 스크립트 다운로드 및 실행권한 부여 
curl -sL https://rock.sh/rockpi-backup -o rockpi-backup.sh
chmod +x rockpi-backup.sh
BASH - USB 메모리에 이미지 백업 실행 
# USB 메모리가 /mnt 경로에 연결되어 있다고 가정한 명령어입니다.
# 실제 마운트된 경로로 백업 위치를 지정하세요.
sudo ./rockpi-backup.sh -o /mnt/rockpi_backup.img
2

2단계: 데스크탑 설정 (rkdeveloptool 빌드 및 설치)

백업된 이미지를 Rockpi에 다시 라이팅하기 위해 개발용 호스트 PC(Ubuntu / Debian 기준) 환경을 세팅합니다.

BASH - 필수 의존성 라이브러리 설치 
sudo apt-get update
sudo apt-get install -y libudev-dev libusb-1.0-0-dev dh-autoreconf pkg-config libusb-1.0-0
BASH - 툴 소스 빌드 및 시스템 등록 
git clone https://github.com/rockchip-linux/rkdeveloptool
cd rkdeveloptool
autoreconf -i
./configure
make -j$(nproc)
sudo cp rkdeveloptool /usr/local/sbin/
3

3단계: Rockpi 마스크롬(Maskrom) 모드 진입

복구 대상 Rockpi 보드를 호스트 데스크톱에서 인식할 수 있도록 특수 모드로 초기화 부팅합니다.

1

전원 케이블, HDMI, LAN, 일반 USB 등 모든 연결선을 제거합니다.

2

보드의 Maskrom 키(좌측 적색 스위치)를 누른 채 유지합니다.

3

스위치를 누른 상태에서 Type-C OTG 포트를 케이블로 PC와 연결한 뒤 스위치를 뗍니다.

Rock 5B+ Board Layout

* 적색 박스로 감싸진 Maskrom 버튼과 상단 OTG(Type-C) 포트를 확인하세요.

데스크탑에서 USB 연결 감지 여부 확인

BASH - 연결 확인 
lsusb | grep "Fuzhou Rockchip Electronics"

목록에 기기가 검색된다면 Maskrom 진입에 정상적으로 성공한 상태입니다.

4

4단계: 백업 이미지 복구(Flash) 실행

준비된 이미지를 보드의 부트로더 세팅을 거쳐 실제로 내장 eMMC 디렉토리에 전송(Flashing)합니다.

잠깐! 파일 경로 확인

아래 과정은 백업해 두었던 rockpi_backup.img 파일이 존재하며, 해당하는 RK3588용 Loader바이너리가 존재하는 폴더 안에서 셸 명령어를 실행해야 합니다.

BASH - 1. 로더(Loader) 구동 명령 
# 보드가 Maskrom인 경우, rkdeveloptool이 해당 로더(SPL)를 메모리에 상주시켜 통신을 돕도록 실행합니다.
sudo rkdeveloptool db rk3588_spl_loader_v1.15.113.bin
BASH - 2. 백업 이미지 쓰기 (Flash) 
# eMMC 메모리의 시작 블록 주소인 0번에 백업 파일 라이팅을 시작합니다.
sudo rkdeveloptool wl 0 rockpi_backup.img
BASH - 3. 디바이스 리셋 
# 정상 플래싱 이후 보드를 재부팅합니다.
sudo rkdeveloptool rd
5

5단계: 완료 및 부팅 검증

  • PC와 연결되어 있던 USB Type-C OTG 케이블을 분리합니다.
  • 전용 전원 아답터를 전원 포트에 다시 연결하여 장비를 부팅합니다.
  • 기존에 백업했던 DeepX 개발 전용 패키지, 설정 및 유저 데이터가 그대로 보존되어 복구가 무사히 끝났는지 점검합니다.
효율적인 마스터링 팁

백업 이미지 파일 용량이 큰 상태(64GB+)라면, rkdeveloptool wl을 진행하는 과정에서 전송이 수 분 이상 소요될 수 있습니다. 고속 전송이 가능한 정품 USB 3.0 Type-C 케이블 연결을 적극 권장하며, 통신이 끊어지지 않도록 데스크톱 및 보드의 전원 관리에 유의하십시오.
